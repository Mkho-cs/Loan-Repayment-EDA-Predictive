Predictive Modelling¶

Feature Selection¶

The decisions made here are mainly supported by insights gained in the correlation section of eda.ipynb.

Summary of decisions:

  • Payment amount and originally scheduled payment variables will be removed as the target variable is derived from these values.

  • Loan amount will be kept since payment amount and scheduled amount are removed.

  • Loan status is removed due to being a dynamic variable and a contender to the target variable, hence not suitable as a predictor.

  • Lead cost is considered a redundancy as there is lead type.

  • Year is removed as it is not a useful variable, instead loan term would be a better variable related to time, but data on loan term is not found.

Cost Complexity Pruning¶

Previously, we observed that vanilla decision trees and random forest models performs rather poorly due to overfitting. Hence, these models can be improved by pruning.
Minimal cost complexity pruning recursively finds the node with 'weakest link' (smallest effective alpha) to remove.

Pruned Decision Tree¶

In [23]:
# Load model, get cost complexity alphas and impurities
vanilla_dt = load(open(os.path.join(stratified_70_30_path, 'vanilla_dt.pkl'), 'rb'))
complexity_path = vanilla_dt.cost_complexity_pruning_path(x_train, y_train)
ccp_a, impurities = complexity_path.ccp_alphas, complexity_path.impurities
print("Number of ccp alphas (same as number of subtrees):", len(ccp_a))

Number of ccp alphas (same as number of subtrees): 1140

Notes on cost complexity pruning:

  • Since there is a large number of alphas, experimenting with each of them will take lots of time.

  • At the last index, there is only one node, hence it is not viable.

  • Using the first few indices doesn't prune the tree that much, so its not viable either.

  • Hence, lets use an alpha thats in the middle or close to the end of the list (i.e. index -1).
In [26]:
alpha = ccp_a[1100] # An alpha that prunes most of the tree
pruned_dt = DecisionTreeClassifier(ccp_alpha=alpha, random_state=12345)
pruned_dt.fit(x_train, y_train)
train_roc_auc, val_roc_auc = mod.evaluate_roc_auc(pruned_dt, train, val)
print("CCP alpha:", alpha)
print("Training score:", train_roc_auc)
print("Validation score: ", val_roc_auc)
mod.plot_roc(pruned_dt, val, "Pruned Decision Tree")

Key insights on pruned decision tree:

  • The pruned tree performs much better than the vanilla tree on the validation set, about 20% increase in AUC score.

  • Training score decreases, approximately proportional to the increase in validation score; now both are similar in comparison.

  • Now the pruned tree shows the ability to generalize compared to the vanilla tree that overfits very badly.

  • Overall, the increase in performance is astounding, and model size is decreased as well.

  • This pruning technique shows that a simple model can be powerful enough with proper optimization .
